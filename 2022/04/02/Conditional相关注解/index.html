Conditional相关注解

笔记
SPRING
发布日期:   2022-04-02

Conditional

它的作用是按照一定的条件进行判断，满足条件给容器注册 bean。条件就是注解中 Condition 的实现类的实现方法 matches 的返回值（true 或者 false），可以有多个条件，必须同时为 true 才会注册 bean，该注解也可以用在配置类上，来决定了一批 bean 是否注入。

代码演示：

//配置类

@Configuration("beanConfig2")
public class BeanConfig {
    @Conditional(WindowsCondition.class)
    @Bean(name = "bill")
    public Person person1() {
        return new Person("bill gate", 44);
    
    }

    @Bean(name = "linus")
    @Conditional(LinuxCondition.class)
    public Person person2() {
        return new Person("linus", 55);
    }
}

//WindowsCondition
public class WindowsCondition implements Condition {
        @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        if(context.getEnvironment().getProperty("os.name").equals("windows")) {
            return true;
        }
        return false;
    }
}

//LinuxCondition
public class LinuxCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        if (context.getEnvironment().getProperty("os.name").equals("Mac OS X")) {
            return true;
        }
        return false;
    }
}



可以看到只注入了 linus 的 Bean，bill 的 Bean 并没有注入。

ConditionalOnBean

要求额外的 bean 存在时，才会创建这个 bean；

如我提供了一个 bean 名为 RedisOperBean，用于封装 redis 相关的操作；但是我这个 bean 需要依赖 restTemplate 这个 bean，只有当应用引入了 redis 的相关依赖，并存在 RestTemplatebean 的时候，RedisOperBeanbean 才会生效。
代码举例：

@Configuration
public class BeanConfiguration1 {
    @Bean
    public Bean1 bean1(){
        return new Bean1();
    }
}

@Configuration
public class BeanConfiguration2 {

    @Bean
    @ConditionalOnBean(Bean1.class)
    public Bean2 bean2(){
        return new Bean2();
    }
}

只有 Bean1 初始化了，Bean2 才能进行初始化

如果换一下位置

@Configuration
public class BeanConfiguration1 {
    @Bean
    @ConditionalOnBean(Bean2.class)
    public Bean1 bean1(){
        return new Bean1();
    }
}

@Configuration
public class BeanConfiguration2 {
    @Bean
    public Bean2 bean2(){
        return new Bean2();
    }
}

可以发现 Bean1 的初始化失败了。主要是因为 BeanConfiguration1 比 BeanConfiguration2 的配置类要更先加载，初始化 Bean1 的时候发现容器中没有 Bean2，所以 Bean1 就没有初始化。

解决方案

  • 通过 ConditionalOnClass(@ConditionalOnClass 通常与@Configuration 结合使用，意思是当 classpath 中存在某类时满足条件)。
  • 通过 AutoConfigureBefore 或者 AutoConfigureAfter 注解，但是请注意：**@AutoConfigureBefore 和@AutoConfigureAfter 这两个注解是针对使用 spring.factories 加载的配置(也就是说注解中的 Bean 是必须通过 spring.factories 来加载)。如果是自定义的实现类是在项目中被自动扫描后进行配置的，注解不会生效。**
//注意两个类都没有配置@Configuration注解
public class BeanConfiguration1 {
    @Bean
    @ConditionalOnBean(Bean2.class)
    public Bean1 bean1(){
        return new Bean1();
    }
}

@AutoConfigureBefore(BeanConfiguration1.class)
public class BeanConfiguration2 {
    @Bean
    public Bean2 bean2(){
        return new Bean2();
    }
}

ConditionalOnMissingBean

@ConditionalOnMissingBean，它是修饰 bean 的一个注解，主要实现的是，如果当前没有这个类型的 Bean（注解的值），就会使用这个注解使用的 Bean。

@Configuration
public class BeanConfig {
    @Bean
    @ConditionalOnMissingBean(Computer.class)
    public Computer c1(){
        return new Computer();
    }
    @Bean
    public Computer c2() {
        return new Computer();
    }
}


打印这两个 Bean：

因为 c1 的时候还并没有 Computer 的 Bean 注入，所以 c1 也注入成功了。
如果换一下：

@Configuration
public class BeanConfig {
    @Bean
    public Computer c1(){
        return new Computer();
    }
    @Bean
    @ConditionalOnMissingBean(Computer.class)
    public Computer c2() {
        return new Computer();
    }
}

结果：

因为 c2 注入的时候发现已经有了 Computer 的 Bean 了，所以 c2 这个 Bean 是不会注入的。

ConditionalOnProperty

场景：项目中自带 mongodb，但是 mongodb 并不是项目中所必须要有的中间件，所以 mongodb 的配置需要根据实际需要来生效或不生效的。

使用 ConditionalOnProperty 就可以通过 yml 中配置的属性来达到这个效果。

  • value:获取对应 property 名称的值，与 name 不可同时使用
  • prefix：属性名称的前缀
  • name：数组，配置属性完整名称或部分名称，可与 prefix 组合使用，组成完整的配置属性名称，与 value 不可同时使用
  • havingValue：比较获取到的属性值与 havingValue 给定的值是否相同，相同才加载配置
  • matchIfMissing：缺少该配置属性时是否可以加载。如果为 true，没有该配置属性时也会正常加载；反之则不会生效

配置类使用：

/**
 * @author qianxin
 * @date 2022/2/18
 * ConditionalOnProperty
 *     检查hscs.mongodb.enable-flag属性是否为false，如果为false，则加载当前的Bean，matchIfMissing默认为false 未进行属性配置，则自动配置不生效
 */
@Configuration
@EnableAutoConfiguration(exclude = {MongoAutoConfiguration.class, MongoDataAutoConfiguration.class})
@ConditionalOnProperty(prefix = "hscs.mongodb", name = "enable-flag", havingValue = "false", matchIfMissing = true)
public class ExcludeMongoConfig {
}

//当是oracle数据库时，才会加载Bean
@Bean
@ConditionalOnMissingBean(OracleDialectHandler.class)
@ConditionalOnExpression("'${spring.datasource.url:}'.contains('oracle')")
public QueryHandler oracleDialectHandler() {
    return new OracleDialectHandler();
}
