InheritableThreadLocal

多线程
发布日期:   2022-05-21

由于 ThreadLocal 设计之初就是为了绑定当前线程，如果希望当前线程的 ThreadLocal 能够被子线程使用，实现方式就会相当困难（需要用户自己在代码中传递）。在此背景下，InheritableThreadLocal 应运而生。

Thread 类中还有一个变量：

inheritableThreadLocals 即主要存储可自动向子线程中传递 ThreadLocal.ThreadLocalMap

/**
 * 默认情况下，设置inheritThreadLocals可传递
 */
private void init(ThreadGroup g, Runnable target, String name,
                  long stackSize) {
    init(g, target, name, stackSize, null, true);
}

采用默认方式产生子线程时，inheritThreadLocals=true；若此时父线程 inheritableThreadLocals 不为空，则将父线程 inheritableThreadLocals 传递至子线程。

这个过程只在启动子线程的时候，之后两个线程在 ThreadLocal 中的值就是独立的，互不影响。

源码解析

继承自 ThreadLocal，并且重写了父类的方法：createMap(),getMap(),childValue();

应用

全链路跟踪(压测)必备基础组件之线程上下文“三剑客”

但是一般使用线程的时候会使用线程池，线程池的线程复用技术会导致线程在一开始启动的时候 ThreadLocal 就被继承了，之后复用的时候这个值还是一开始的值，看一个例子

private static InheritableThreadLocal<String> local = new InheritableThreadLocal<>();
    public static void main(String[] args) throws InterruptedException {
        local.set("FIRST_VALUE");
        ExecutorService executorService = Executors.newFixedThreadPool(2);
        executorService.submit(()->{
            System.out.println(Thread.currentThread().getName()+":"+local.get());
        });
        Thread.sleep(1000);
        local.set("SECOND_VALUE");
        for(int i=0;i<4;i++) {
            executorService.submit(()->{
                System.out.println(Thread.currentThread().getName()+":"+local.get());
            });
        }
        executorService.shutdown();
    }

运行结果：

可以看到，线程 1 在被第一次设置之后，主线程再次更新 value，线程 1 的值还是不会变。

想要解决这个问题，通常的解决方案就是阿里的 transmittable-thread-local

transmittable-thread-local

在使用线程池等会池化复用线程的执行组件情况下，提供 ThreadLocal 值的传递功能，解决异步执行时上下文传递的问题。

JDK 的 InheritableThreadLocal 类可以完成父线程到子线程的值传递。但对于使用线程池等会池化复用线程的执行组件的情况，线程由线程池创建好，并且线程是池化起来反复使用的；这时父子线程关系的 ThreadLocal 值传递已经没有意义，应用需要的实际上是把 任务提交给线程池时的 ThreadLocal 值传递到 任务执行时。

如何解决上述问题

修饰 Runnable 和 Callable

使用 TtlRunnable 和 TtlCallable 来修饰传入线程池的 Runnable 和 Callable。

static class RunnableTask implements Runnable{
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName()+":"+context.get());
    }
}

private static TransmittableThreadLocal<String> context = new TransmittableThreadLocal<>();
    public static void main1(String[] args) throws InterruptedException {
        context.set("value-set-in-parent");
        ExecutorService executorService = Executors.newFixedThreadPool(2);
        Runnable task = new RunnableTask();
        Runnable ttlRunnable = TtlRunnable.get(task);
        executorService.execute(ttlRunnable);
        Thread.sleep(1000);
        context.set("value-change");
        for (int i=0;i<4;i++){

            Runnable task2 = new RunnableTask();
            Runnable ttlRunnable2 = TtlRunnable.get(task);
            executorService.execute(ttlRunnable2);
        }
        executorService.shutdown();

    }

运行结果

可以看到，主线程重新设置之后，线程 1 里面 ThreadLocal 的值也变掉了

修饰线程池

省去每次 Runnable 和 Callable 传入线程池时的修饰，这个逻辑可以在线程池中完成。通过工具类 com.alibaba.ttl.threadpool.TtlExecutors 完成，有下面的方法：

  • getTtlExecutor：修饰接口 Executor

  • getTtlExecutorService：修饰接口 ExecutorService

  • getTtlScheduledExecutorService：修饰接口 ScheduledExecutorService

public static void main(String[] args) throws InterruptedException {
        context.set("value-set-in-parent");
        ExecutorService executorService = Executors.newFixedThreadPool(2);
        executorService = TtlExecutors.getTtlExecutorService(executorService);
        executorService.execute(new RunnableTask());
        Thread.sleep(1000);
        context.set("value-set-in-parent2");
        for (int i=0;i<4;i++){
            executorService.execute(new RunnableTask());
        }
        executorService.shutdown();
    }

运行结果：

线程 1 里面 ThreadLocal 的值也变掉了
